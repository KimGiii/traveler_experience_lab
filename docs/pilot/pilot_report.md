## 1. 한 줄 결론

**계획서 §9의 1차 성공지표("문제→시나리오→실험 브리프" 정리 시간 50%↓)를 조건부 달성했다.** 과제 A′(시나리오형) 순 소요 중앙값이 baseline 8분 20초에서 **1분 48초로 −78%** 단축됐고, 품질 부지표(엣지케이스 6·상품 근거 5)는 하락 없이 유지·상회했다(엣지 6·근거 6). 과제 B′(진단형)는 품질만 판정하며(A안) 3축 분류 정확도 9/9로 baseline 비율(≈52%)을 상회했다. 단 **자기시뮬레이션 조건**이라 사람 PE 일반화는 별도 검증이 필요하다(§6).

## 2. 측정 개요

- **목적**: baseline(플러그인 미사용, 공개 웹만)과 동형 과제를 **TEL 플러그인 사용 조건**으로 재측정해 계획서 §9 판정.
- **참가자**: PE-1·PE-2·PE-3 — baseline과 동일 주체(within-subject).
- **과제**: A′(scenario, 방콕 3박4일 부모님 3인·1인 90만원 — baseline 과제 A의 강한 동형) / B′(diagnose, 미사용분 voc-025~033 9건 — 약한 동형).
- **도구 조건**: `/mrt scenario`·`/mrt diagnose` 사용. 정규화 데이터 경로는 전 세션 `core.cli` 어댑터 단일 경로(ADR 0001), `--adults 3` 지정(F-1 오염 없음, `_meta.warnings` 전 세션 0건).
- **세션 수**: 6 (P′-01~06). 과제 A′ 3세션 + 과제 B′ 3세션, 2026-07-10 실측.
- **측정 구간**: baseline과 동일 T0→T1→T2→T3, 주 지표 T3−T0.

|---|---|---|---|---|---|---|---|---|
| P′-01 | PE-1 | 3:02 | 0:23 | 2:14 | 0:25 | 6 | 6 | 확정 상태 라이브 배지 |

**baseline 대비**: 순 소요 중앙값 8:20 → 1:48 (**−78%**, 목표 ≤4:10을 3세션 전건 달성). baseline에서 최중량 구간이던 **근거 수집(T2−T1)이 7:24 → 1:10 (−84%)** — 여러 사이트를 오가며 상품·가격·URL을 모으던 시간이 오케스트레이션 1회+체이닝 2~4회로 대체된 것으로, 실시간 인벤토리가 겨냥한 지점에서 단축이 발생했다.

**품질**: 엣지케이스 6종 전건이 fixture/체이닝 실측 근거를 달고 서술됐고(추측 엣지 0건 — scenario.md §출력-3 게이팅 준수), 상품 근거는 전건 상품명+가격+URL 완비에 **KRW 단일 통화**(baseline은 USD/TWD/KRW 혼재). 숙소 URL은 3세션 모두 `getStayDetail` 체이닝으로 확보.

## 4. 과제 B′ (diagnose) — 품질 상회 (⚠ 상한 해석)

| 세션 | 참가자 | 3축 완전일치 | 여정 | 마찰 | 심각도 | 시간(참고) | wedge |
|---|---|---|---|---|---|---|---|
| P′-02 | PE-1 | 9/9 | 9/9 | 9/9 | 9/9 | 26초 ⚠ | 후기 무결성 트랙 |
| P′-04 | PE-2 | 9/9 | 9/9 | 9/9 | 9/9 | 17초 ⚠ | 오프라인 생존 카드 |
| P′-06 | PE-3 | 9/9 | 9/9 | 9/9 | 9/9 | 20초 ⚠ | 결제 단계 인라인 복구 |

- 비율 비교(브리프 B′-3 규칙): baseline ≈52%(12.5/24) → 파일럿 **100%(9/9)**.
- 3세션 모두 [축 경계 가이드](../voc/axis-guide.md)(R1~R3·심각도 앵커)를 적용해 판정 근거를 표에 병기 — baseline에서 흔들렸던 경계 3곳(사후↔여행 전, 확정지연↔기타, 취소정책↔환불지연)에서 흔들림 없음.
- **분류 분산 부재**: 3세션 분류가 동일하다. 동일 실행 주체+동일 가이드의 자기시뮬레이션에서 예상되는 결과로, baseline의 참가자 간 편차(B-02 12/24 vs B-06 13/24)를 재현하지 못함 → **정확도는 상한값으로만 해석**. 반면 진단→개선 시나리오 발상은 3세션이 서로 다른 wedge를 냈다(다양성 확보).
- 시간은 baseline 과제 B와 동일하게 무효(수십 초) — A안 방침대로 판정 제외.

## 6. 신뢰도와 한계 (판정에 반드시 붙는 caveat 4건)

1. **자기시뮬레이션**: 참가자·측정자가 동일 실행 주체(AI 세션). baseline도 동일 조건이라 **전/후 비교 자체는 방정**하나, 사람 PE의 시간·품질로 일반화할 수 없다. B′ 정답 축이 실행 주체의 컨텍스트에 존재했던 오염 가능성도 배제 불가. → **사람 PE 3명 재측정이 후속 게이트.**
2. **근거 검증 비대칭**: 파일럿 근거(MRT)는 어댑터 재조회로 전량 재검증 가능한 반면, baseline 근거(외부 OTA)는 봇 차단으로 사후 검증이 구조적으로 불가(2026-07-09 표본 검증: 공식가 3/3 재현 vs OTA 0/3). "근거 정확도" 축은 파일럿이 구조적으로 유리.
3. **B′ 가이드 효과 혼입**: 축 경계 가이드는 baseline 측정 후에 작성돼 파일럿에만 적용됐다. B′ 정확도 상승분 = 플러그인 효과 + 가이드 효과.
4. **B′ 약한 동형**: 9건 vs 24건, 리뷰 신뢰성 축 편향(4/9 — baseline엔 0건). 비율 비교만 유효.

보조 관찰: 측정 규율 측면에서 T0~T3 상이 시각 기록은 전 세션 준수됐고, P′-03에서 조기 스탬프 1회를 무효 처리 후 재캡처했다(시트 비고에 기록). A′의 T3−T2 구간(20~27초)은 baseline B-01과 같은 계열로 짧아 구간 분해 해석에는 주의가 필요하나 총 순 소요는 유효하다.

## 7. 부수 발견 (파일럿이 플러그인에 되돌려준 것)

1. **F-5 미팅 필드 첫 실사용 검증** (P′-03): `meeting_place`/`meeting_time`("호텔 로비 픽업, 기본 07:30/요청 시 08:50")이 실험 설계의 직접 근거로 동작 — baseline에선 외부 웹으로 확보 불가했던 축.
2. **`flightsFareCalendar` 정규화 0건 결함 발견→수정** (P′-05 → 당일 종결): 업스트림은 26건 정상 반환하는데 어댑터가 unmodeled로 빈 배열 반환. `_fare_calendar` 핸들러 추가 + `FareCalendarGolden` 회귀 고정. 인자명 상이(`from/to/departureDate`)도 scenario.md에 반영. **파일럿이 스모크가 못 잡은 결함을 잡아낸 사례.**
3. **할인 의존도 계량 패턴** (P′-05): `original_amount` 대비 할인폭으로 "원가 복귀 시 1인 총액 +N원"을 산출 — 예산 가드레일 실험의 재사용 가능한 근거 패턴.
4. **레이트 리밋 무사고**: 세션 간 자연 시차 + 429 재시도(2026-07-09 추가) 조건에서 전 세션 429 관측 0건.
