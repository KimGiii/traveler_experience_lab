## [T1 | 00:44:28] 문제 정의

## [T3 | 00:44:44] 개선 시나리오 + 실험/QA (→ `/mrt scenario` 핸드오프)

**개선 시나리오 1건 — "후기 무결성 트랙"**: 후기가 삭제·수정 잠금·개별 연락의 대상이 되는 구조를 겨냥해, (a) 파트너 신고로 후기가 내려가면 "신고로 검토 중" 상태를 상품 평점 옆에 노출, (b) 답글 이후에도 여행자 후기 수정권 보존(수정 이력 공개로 균형), (c) 48시간 제한을 "여행 종료 후 14일"로 완화. 근거 VoC: 26·27·28·29.

**scenario 핸드오프 브리프**: 목적지 자유(리뷰 표본이 큰 도시 권장, 예: 방콕) / 페르소나 "후기를 보고 투어를 고르는 신중한 여행자" / wedge: 상품 상세의 평점·리뷰수·후기 상태 표기가 선택에 충분한 정보를 주는가 — TNA 후보의 `rating`·`review_count`(미노출 사례 P′-01 실측) 검증 포함.

**실험 초안**: 실험군에 "검토 중 후기 n건" 배지 + 후기 수정 이력 공개. 가설: 평점 신뢰가 회복되면 고평점 상품 의존이 줄고 리뷰 정독·전환의 질이 오른다. 지표: 상품 상세→결제 전환율, 후기 신고율, 후기 관련 CS 문의, 저평점 상품의 부당 이탈. 가드레일: 파트너 분쟁 접수량. 실패조건: 전환 하락 + 분쟁 급증.

**QA 확인 항목**: 신고-삭제-복원 상태머신 전이 전수, 답글 후 수정 시 이력 표시, 48h→14일 경계값(13일 23시/14일 0시), 모바일 후기 작성 화면에서 상태 배지 노출, 비식별 처리(후기 작성자 정보 미노출).
