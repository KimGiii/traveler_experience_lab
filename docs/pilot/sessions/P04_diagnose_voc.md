## [T1 | 00:49:57] 문제 정의

가장 위험한 마찰은 **여행 중 연결 단절의 삼중 구조**(25·31·32): 가이드와 연락이 끊겨도 긴급 연락망이 일반 연락처와 동일해 중재가 불가하고(25), eSIM 안내 페이지 오류로 출국 직전 설치 정보를 못 보며(31), 유심이 불량이면 데이터 없이 도움말에 접근해야 하는 역설(32)에 빠진다 — 셋 다 "통신이 필요한 순간에 통신이 무너지는" 동일 패턴이다. 타깃은 여행 중 문제를 혼자 해결해야 하는 여행자(특히 초행·고령 동반). 성공 기준: 오프라인 폴백 경로 확보와 긴급 연락 실패율 감소.

## [T3 | 00:50:08] 개선 시나리오 + 실험/QA (→ `/mrt scenario` 핸드오프)

**개선 시나리오 1건 — "오프라인 생존 카드"**: 예약 확정 시점에 ① 투어 미팅 장소/시간(`meeting_place`/`meeting_time` — P′-03에서 실데이터 확보 검증됨) ② 파트너 연락처(일반/긴급 구분 상태 포함) ③ eSIM/유심 설치·장애 조치 가이드를 **오프라인 저장 가능한 단일 카드**로 묶어 발급. 데이터 없이도 열람 가능(31·32 역설 해소), 긴급 연락처가 일반과 동일하면 카드에 "플랫폼 채팅 우선" 대체 경로 표기(25).

**scenario 핸드오프 브리프**: 목적지 방콕 / 페르소나 "부모님 동반, 여행 중 문제를 자가 해결해야 하는 예약 담당자" / wedge: 확정 후 제공 정보(미팅·연락처)가 데이터 계층에서 어디까지 채워지는가 — `getTnaDetail` 미팅 필드 커버리지 확인 포함.

**실험 초안**: 실험군에 확정 메일+마이페이지 "오프라인 생존 카드" 제공. 가설: 여행 중 긴급 CS 인입(특히 야간)과 가이드 연락 실패 문의가 감소. 지표: 여행 중 CS 인입률, 긴급 채널 전환 시간, 카드 저장률/열람률. 가드레일: 카드 정보-실제 불일치 신고. 실패조건: 저장률 <10% 또는 불일치 신고 증가.

**QA 확인 항목**: 카드 오프라인 열람(비행기 모드) E2E, 미팅 정보 없는 상품의 카드 폴백 문구, 긴급=일반 연락처 동일 감지 로직, eSIM 안내 링크 생존성(31 재발 방지), 다국어(현지 제시용 태국어 문구), 고령 사용자 글자 크기.
