## [T3 | 00:48:10] 엣지케이스 + 실험/QA 초안

**엣지케이스 6종 (전건 데이터 근거):**
- **취소/환불**: 숙소 `free_cancellation=True` 실측 ↔ 단독투어는 `cancellation_note` 빈 값 — 취소 규정이 예약 전 응답에 없어 결제 화면 의존. 카테고리 간 비대칭.
- **재고없음**: 숙소 `available=True`·투어 11-08 가용은 조회 시점 값. 단독투어(4인 기준 요금)는 인원 구성 변경 시 재확인 필요.
- **날짜불일치**: `getTnaOptions`는 날짜 단위 확인(11-08 실측) — 일정 하루 이동 시 재조회 없이는 보장 없음. 미팅 시간도 "호텔 위치에 따라 10~20분 상이" 조건부.
- **가격변동**: 숙소 원가 338,044→273,913원 할인 관측 — 종료 시 1인 총액 +64,131원/박×3. 항공 fixture `price_change`.
- **리뷰신뢰성**: 단독투어 4.6/1,151건·숙소 4.7/545건은 표본 있음. 단 getTnaDetail 첫 제목이 상품명이 아닌 **옵션명**("4인 단독출발 1인 요금")으로 반환 — 상세 화면 신뢰도 이슈(field_notes §4 재확인).
- **모바일**: 전 예약 URL 모바일 웹 — 미팅 장소/시간처럼 긴 조건 텍스트가 모바일에서 접히면 부모님 동행자가 07:30 출발을 놓칠 위험.

**실험 초안 — "미팅 정보 재확인 카드(T-24h)"**: 투어 전날, 예약 확정 건의 `meeting_place`/`meeting_time`을 푸시+카드로 재노출하고 "출발 시간 조정 요청" CTA 연결(이 상품은 08:50 요청 가능 실측). 가설: 부모님 동반 여행자의 노쇼·집합 혼선 감소. 지표: 투어 노쇼율, 미팅 관련 CS 문의, 카드 열람→시간조정 요청 전환. 가드레일: 조정 요청 폭주로 인한 파트너 응답 지연. 실패조건: 노쇼 개선 없이 파트너 분쟁만 증가. (voc-024 노쇼 책임 전가와 직결)

**QA 확인 항목**: 3인 인원 전 URL 유지, 미팅 시간 조건부 문구("호텔 위치 따라 상이") 원문 그대로 노출(요약 왜곡 금지), 무료취소 배지-실제 규정 일치, 11-06~09 가격 고정 재현, 상세 화면 옵션명이 상품명으로 오표기되지 않는지, 모바일에서 미팅 정보 접힘 없음, 취소정책 부재 상품에 "결제 전 확인" 표기.
