## [T3 | 00:52:57] 엣지케이스 + 실험/QA 초안

**엣지케이스 6종 (전건 데이터 근거):**
- **취소/환불**: 숙소 `free_cancellation=True` 실측 ↔ 조인투어 취소정책 문구 예약 전 응답에 없음 — 예산 방어 관점에서 "취소 시 회수 가능액"이 상품별로 불투명.
- **재고없음**: 숙소·투어 가용은 조회 시점 값. 조인투어는 정원 구조라 3인 동시 확정이 임박 시점에 흔들릴 수 있음.
- **날짜불일치**: 투어 가용은 11-07 단위 실측 — 날짜 이동 시 재조회 필요. `flightsFareCalendar`로 날짜별 운임 비교를 시도했으나 **정규화 후보 0건**(1차 호출은 인자명 상이 `from/to/departureDate`로 실패) — 날짜 대안 탐색이 데이터 계층에서 아직 약함(실측).
- **가격변동**: 숙소 213,762→173,040원(19%) 할인 실측 — 만료 시 1인 총액 +40,722원. 항공 fixture `price_change` 플래그. 총액의 할인 의존도가 계량됨.
- **리뷰신뢰성**: 숙소 4.6/1,029건 표본 충분 ↔ 조인투어 ⭐4.8 **리뷰수 미노출**(`review_count=null`) — 최저가 대안일수록 리뷰 근거가 얇은 비대칭.
- **모바일**: 전 예약 URL 모바일 웹 — 1인 총액·할인 만료 정보가 모바일 카드에서 유지되는지가 예산 방어의 UX 관건.

**실험 초안 — "1인 총액 고정 바 + 할인 만료 경고"**: 장바구니/브리프 화면 상단에 항공+숙소+투어 1인 환산 총액을 고정 노출하고, 할인가 항목에 "원가 복귀 시 +40,722원" 식 변동폭 명시. 가설: 예산 민감 여행자의 결제 진입률 상승·가격변동 관련 문의/이탈 감소. 지표: 결제 진입률, 구성 완료율(3개 도메인 모두 담기), 가격변동 CS 문의율. 가드레일: 총액 계산 오차. 실패조건: 총액 표시 오차 >1% 또는 경고로 인한 이탈 증가.

**QA 확인 항목**: 1인 환산식(차량 단위·객실 단위·인 단위 혼합) 정확성, 3인 인원 전 URL 유지, 할인 만료 시 총액 자동 갱신, 조인/단독 가격 비교 표기, fareCalendar 미지원 구간 폴백 문구, 모바일 고정 바 겹침 없음, 11-06~09 날짜 고정 재현.
