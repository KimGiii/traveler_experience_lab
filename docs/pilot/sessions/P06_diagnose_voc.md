## [T1 | 00:54:31] 문제 정의

예약·결제 국면에서 여행자가 손해를 감지하고도 복구할 수 없는 마찰에 주목한다: 할인코드가 결제 단계에서 오류로 막히고(30), 후기 생태계가 파트너에 기울어 가격 대비 품질 판단의 근거(평점)를 믿을 수 없다(26~29). 예산 민감 여행자에게 "할인 실패"와 "평점 인플레"는 같은 문제의 양면 — **지불 판단의 근거가 결제 순간에 무너진다**. 타깃은 프로모션·평점을 근거로 상품을 고르는 가격 민감 여행자. 성공 기준: 결제 단계 코드 오류의 인라인 복구율과 평점 신뢰 지표 개선.

## [T3 | 00:54:44] 개선 시나리오 + 실험/QA (→ `/mrt scenario` 핸드오프)

**개선 시나리오 1건 — "결제 단계 인라인 복구"**: 할인코드 적용 오류(30) 시 결제 화면 이탈 없이 ① 코드 유효성 원인(만료/조건 미충족/시스템 오류) 즉시 표기 ② 시스템 오류면 자동 재시도+대체 코드 제안 ③ 실패 지속 시 "코드 없이 결제 후 차액 보전 요청" 경로 제공. 26~29(평점 신뢰)는 별도 트랙(P′-02 후기 무결성)과 중복 회피를 위해 참고 연결만.

**scenario 핸드오프 브리프**: 목적지 방콕 / 페르소나 "프로모션·최저가 중심으로 3인 예산을 맞추는 예약 담당자" / wedge: 결제 직전 가격 무결성 — 검색가→상세가→결제가 일치 검증(P′-05의 할인 의존 계량과 연결), 할인 상품의 원가 복귀 시나리오 포함.

**실험 초안**: 실험군에 코드 오류 인라인 복구 UI. 가설: 결제 단계 코드 오류 경험자의 결제 완료율이 대조군 대비 상승, "할인 적용 안 됨" CS 문의 감소. 지표: 코드 오류 후 결제 완료율, 오류 원인별 분포, CS 문의율, 차액 보전 요청률. 가드레일: 대체 코드 남발로 인한 마진 손실. 실패조건: 완료율 개선 없이 보전 요청만 증가.

**QA 확인 항목**: 코드 오류 원인 3분류 정확성, 재시도 멱등성(중복 결제 방지), 차액 보전 경로 E2E, 모바일 결제 화면 인라인 메시지 노출, 만료 코드 경계값(자정), 비로그인→로그인 전환 시 코드 유지.
