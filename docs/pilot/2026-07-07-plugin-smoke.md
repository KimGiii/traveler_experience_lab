## 발견 사항 (우선순위순)

### F-1 · `--adults` 기본값 2의 함정 (혼자 여행 오염) — 수정 필요
- 증상: 스페인(혼자) 런에서 검색·`getStayDetail` 체이닝 모두 성인 2로 실행, 예약 URL에 `adultCount=2` 고정.
- 원인: `core.cli scenario`의 `--adults` 기본값 2(`core/cli.py:377`). `--companions`는 자유 텍스트라 인원과 연결되지 않고, `commands/scenario.md`도 `--adults` 지정을 지시하지 않음. 전파 자체(`planning.py:98,121`)는 일관됨 — **입구의 침묵 기본값** 문제.
- 대응 방향: scenario.md 실행 예시에 `--adults` 필수 표기 + 동행자→인원 매핑 지시. (CLI에서 미지정 시 경고 출력도 후보)

### F-2 · 숙소 검색가 vs 상세가 2.0× 불일치 (가격 의미론)
- 증상: 그라나다 게스트하우스 — 검색 73,739원/박 vs 상세 147,478원/박(**정확히 2배**). 오사카 APA 호텔은 두 값 일치.
- 가설: 상품 유형에 따라 검색가가 1인 기준, 상세가 객실 기준. F-1과 무관하게 존재하는 upstream 의미론 차이로 추정.
- 대응 방향: 게스트하우스/민박 유형 3건 샘플로 검증 후 `field_notes.md` §3에 가격 기준 명시. scenario 출력에서 검색가 인용 시 주의 문구.

### F-3 · 국제선 일부 응답에서 정규화 가격 미반환
- 증상: 스페인(BCN 왕복 환승)·다낭(DAD 왕복) 런에서 항공 후보 전건 가격 null + `price_change` 플래그. 후쿠오카(FUK)는 정상 반환.
- 가설: 노선/왕복 구성에 따른 응답 변형을 `adapters.py` flights 추출이 커버하지 못함.
- 대응 방향: BCN/DAD 케이스 raw fixture 확보(`core.cli call` 디버깅) → 정규화 경로 보강 + 골든 추가.

### F-4 · `getTnaDetail` 상품 불일치 응답 (다낭 런 관측)
- 요청한 gid와 다른 상품의 상세가 반환된 것으로 관측됨. 해당 세션에서 조사 태스크로 분리됨. 재현 fixture 확보 필요.

### F-5 · 미팅 포인트/가이드 연락처 필드 부재 (기획 발견)
- `ProductCandidate` 정규화 스키마에 미팅 장소·가이드 연락처·긴급 연락 필드가 없음. voc-005/025(연락망 부재)의 **데이터 계층 대응물** — "가이드 긴급 연락처 자동 노출" 류 실험의 0단계 선행 과제.
- 단, MCP 응답 원문에 해당 필드가 존재하는지부터 미확인 — `getTnaDetail` copy_text 파싱 여지 조사 후 어댑터 확장 여부 판단.
