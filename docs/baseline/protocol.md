# Baseline 측정 프로토콜 — "문제 → 시나리오 → 실험 브리프" 소요시간

- 상태: 준비 완료 (실측 대기 — PE 참여 필요)
- 작성일: 2026-07-06
- 관련: [계획서 §8-4/§9](../../traveler_experience_lab_plan.md), 기록 시트 [`baseline_sheet.md`](baseline_sheet.md)

## 왜 지금 측정하는가

계획서 §9의 1차 성공지표는 **"문제→이슈/PRD 정리 시간 50%↓"** 이며, baseline 없이는 파일럿 후 판정이 불가능하다.
**플러그인 사용 전에** 측정해야 오염되지 않는다 — 파일럿 시작 후에는 baseline을 소급 측정할 수 없다.

## 측정 대상

- **참가자**: PE 2~3명 (파일럿 참가 예정자와 동일 인원 권장 — within-subject 비교).
- **통제 조건**: TEL 플러그인(`/mrt scenario`, `/mrt diagnose`) **미사용**. 그 외 평소 쓰는 모든 도구(Cursor/Codex/범용 LLM/사내 위키) **허용**.
  - 근거: 플러그인의 차별점은 "LLM 사용 여부"가 아니라 **실시간 인벤토리 + 실제 VoC 근거화**다. 범용 LLM을 금지하면 비교가 불공정해진다.

## 표준 과제 2종 (파일럿 때 동형 과제로 재측정)

| 과제 | 내용 | 대응 명령 |
|---|---|---|
| **A. scenario형** | 주어진 목적지/기간/페르소나로 "실제 상품 근거가 있는 여행자 시나리오 + 도메인 엣지케이스 + 실험/QA 초안" 브리프 작성 | `/mrt scenario` |
| **B. diagnose형** | 비식별 VoC 요약 20~30건([`fixtures/voc/voc_samples.json`](../../fixtures/voc/voc_samples.json)에서 발췌)을 여정 단계×마찰 유형×심각도로 구조화하고 개선 시나리오 1건 도출 | `/mrt diagnose` |

과제 브리프 예시(A): "오사카 3박 4일, 가족(초등 자녀 동반), 예산 1인 80만원. 항공+숙소+TNA 복합 시나리오와 취소/환불·재고·날짜 엣지케이스, 실험 초안까지."

> 파일럿 재측정 시에는 **다른 목적지/VoC 세트**로 동형 과제를 내되 난이도를 맞춘다 (동일 과제 반복 시 학습 효과로 오염).

## 측정 구간 (타임스탬프 4개)

| 지점 | 정의 |
|---|---|
| **T0** | 과제 브리프 수령 |
| **T1** | 문제 정의 완료 (해결할 마찰/타깃 사용자/성공 기준 1문단) |
| **T2** | 시나리오 + 상품 근거 완성 (상품명·가격·URL 인용 포함) |
| **T3** | 실험/QA 초안 포함 최종 브리프 완성 |

- 주 지표: **T3−T0** (총 소요시간).
- 보조 지표: T1−T0 (문제 정의), T2−T1 (근거 수집 — 플러그인이 가장 크게 줄일 구간), T3−T2 (실험/QA 초안).
- 중단(회의 등) 발생 시 중단 시간을 기록하고 순 작업시간으로 보정.

## 품질 부지표 (시간만 재면 안 되는 이유)

빨라져도 품질이 떨어지면 실패다. §9의 나머지 지표와 연결해 산출물에서 함께 카운트한다:

| 부지표 | 세는 법 |
|---|---|
| 도메인 엣지케이스 수 | 취소/환불·재고없음·날짜불일치·가격변동·리뷰신뢰성·모바일 중 브리프에 근거와 함께 명시된 항목 수 |
| 실제 상품 근거 수 | 상품명+가격+URL이 모두 있는 인용 수 (검증 불가한 상상 상품은 0점) |
| 근거 정확도 | 인용된 상품/가격이 실제 조회로 재현되는 비율 (측정자가 사후 표본 검증) |

## 실행 절차

1. 측정자가 과제 브리프를 전달하며 T0 기록.
2. 참가자는 T1/T2/T3 도달 시 셀프 타임스탬프 (메신저 메시지 등 검증 가능한 방식 권장).
3. 산출물(브리프 문서)을 제출받아 품질 부지표 카운트.
4. [`baseline_sheet.md`](baseline_sheet.md)에 기록.
5. PE 2~3명 × 과제 2종 = **4~6회 세션** 확보 시 baseline 확정 (중앙값 사용 — 표본이 작아 평균은 이상치에 취약).

## 판정 규칙 (파일럿 후)

- 파일럿 동형 과제의 T3−T0 **중앙값이 baseline 중앙값의 50% 이하**이면 §9 1차 지표 달성.
- 단, 품질 부지표(엣지케이스 수·상품 근거 수)가 baseline **이상**일 때만 유효.
