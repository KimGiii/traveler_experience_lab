# 파일럿 동형 과제 세트 (플러그인 사용)

- 작성일: 2026-07-08
- 상태: 배포 준비 완료 (파일럿 대기)
- 관련: baseline 과제 [`task_briefs.md`](task_briefs.md) · 결과 [`baseline_report.md`](baseline_report.md) · 프로토콜 [`protocol.md`](protocol.md)

> **파일럿 = 플러그인 사용 조건.** baseline은 플러그인 없이(공개 웹만) 측정했고, 파일럿은 **TEL 플러그인(`/mrt scenario`, `/mrt diagnose`)을 사용**해 동일 구조의 과제를 재측정한다.
> 학습효과 오염을 막기 위해 **목적지·VoC 세트를 baseline과 다르게** 냈고, 난이도는 맞췄다(동행 구조·기간·예산·엣지 표면 동일).
> **within-subject 권장**: baseline에 참여한 PE-1·PE-2·PE-3이 그대로 참여(같은 사람의 전/후 비교가 가장 강력).

---

## 공통 규칙 (baseline과 동일, 조건만 반대)

- **도구**: TEL 플러그인 **사용**(`/mrt scenario`, `/mrt diagnose`). 그 외 평소 도구도 허용.
- **타임스탬프**: T0(브리프 수령)→T1→T2→T3를 각각 `YYYY-MM-DD HH:MM:SS KST`로. ⚠ **네 시각을 서로 다르게** 기록(동일 시각 금지 — baseline에서 반복된 실패). 중단 있으면 구간(분) 별도.
- **기록**: 세션 결과는 [`baseline_sheet.md`](baseline_sheet.md)와 동형 스키마로 별도 파일럿 시트에 남긴다(P-세션 또는 신규 시트).
- **⚠ 레이트 리밋 (2026-07-09 실측)**: MCP 엔드포인트는 무지연 버스트 ~5콜에서 429를 반환한다(60초 대기). 어댑터가 자동 재시도(최대 2회)하지만 그만큼 순 소요가 늘 수 있다 — **PE 3명이 동시에 시작하지 말고 세션 시작을 2~3분 간격**으로 벌린다. 429 재시도 대기가 발생한 세션은 비고에 기록(시간 해석 시 감안).

## 파일럿 과제 A′ (scenario) — 방콕, 부모님 3인

> baseline 과제 A(타이베이 3박4일 부모님 3인)의 **동형**. 목적지만 방콕으로 교체, 동행·기간·예산·페르소나 난이도 동일.

### A′-1. 참가자 브리프

> **당신은 마이리얼트립 PE입니다. `/mrt scenario`를 사용해 아래 여행자의 "실험 브리프"를 작성하세요.**
>
> - 목적지: **방콕(태국)** / 출발지: 인천(ICN, IATA `BKK`)
> - 기간: **2026-11-06 ~ 2026-11-09 (3박 4일)**
> - 동행자: **60대 부모님 2명 + 본인 = 성인 3인**
> - 예산: **1인 90만원** (항공·숙소·투어 합산)
> - 페르소나: 해외여행이 익숙지 않은 부모님과 동행. 더위·긴 도보·새벽 이동을 줄이고 싶고, 예약 "확정"에 민감. 야시장·사원·강변 같은 대표 체험은 놓치고 싶지 않음.
>
> **산출물(순서대로 T0~T3 타임스탬프)**: baseline 과제 A와 동일 — [T1] 문제 정의 / [T2] 시나리오 + 상품 근거(상품명+가격+URL 필수) / [T3] 엣지케이스 6종(취소/환불·재고없음·날짜불일치·가격변동·리뷰신뢰성·모바일) + 실험 1건 + QA.
>
> ⚠ `/mrt scenario`의 `--adults 3`을 반드시 넘긴다(미지정 시 2인 오염). 출력의 `_meta.warnings`가 있으면 그대로 보고.

### A′-2. 측정자 채점 가이드 (baseline과 동일)

- 주 지표 **T3−T0**를 baseline 과제 A 중앙값 **8분 20초**와 비교. **목표 ≤ 4분 10초.**
- 품질: 엣지케이스 수(목표 ≥ 6) · 상품 근거 수(상품명+가격+URL, 목표 ≥ 5). 파일럿은 MRT 상품이라 **근거 정확도 재현이 baseline(외부 웹)보다 유리** — 이 차이를 해석에 반영.

## 파일럿 과제 B′ (diagnose) — 신규 VoC 9건

> baseline 과제 B는 voc-001~024(24건)를 썼다. 파일럿은 **미사용분 voc-025~033(9건)**을 쓴다(학습효과 회피).

### B′-1. 참가자 브리프

> **당신은 마이리얼트립 PE입니다. `/mrt diagnose`를 사용해 아래 VoC 9건을 구조화하고 개선 시나리오 1건을 도출하세요.**
>
> **산출물(순서대로 T0~T3)**:
> - [T1] 문제 정의 (핵심 마찰 + 타깃 + 성공 기준)
> - [T2] **9건 전부**를 여정 단계 × 마찰 유형 × 심각도 표 1개로 분류(클러스터 요약은 표 다음에). 축 값은 baseline과 동일 체계.
> - [T3] 개선 시나리오 1건 + 실험 1건 + QA 확인 항목.

### B′-2. VoC 세트 (voc-025~033, 요약문만)

| # | VoC 요약 |
|---|---|
| 25 | 가이드가 일반/긴급 연락처를 동일하게 등록하면 연락 두절 시 플랫폼 중재가 불가 — 긴급 연락망 구조적 취약점을 파트너 문서가 인정. |
| 26 | 투어사가 부정적 후기를 신고해 삭제시키는 사례 → 평점(4.8~5.0 집중)을 그대로 믿기 어렵다는 커뮤니티 인식. |
| 27 | 파트너가 답글을 달면 여행자가 후기를 수정·삭제할 수 없음 — 후기 통제권 비대칭. |
| 28 | 저평점 후기를 받은 파트너가 여행자에게 개별 연락해 삭제/수정을 요청하는 관행(플랫폼이 '지양' 권고문을 둘 정도). |
| 29 | 후기는 최초 작성 후 48시간 이내에만 수정 가능 — 여행 후 뒤늦게 드러난 문제를 후기에 반영하기 어려움. |
| 30 | 할인코드 적용 오류로 결제 단계에서 막히는 경험담. |
| 31 | eSIM 예약 후 안내 페이지 오류로 설치 정보를 확인 못 함 — 출국 직전 시간 압박과 결합. |
| 32 | 유심 불량 시 슬롯 재장착·재부팅·APN 초기화를 현지에서 직접 수행 — 데이터 없이 도움말 접근이 어려운 역설. |
| 33 | 숙박 플랫폼 전반 오버부킹 분쟁 급증·책임 미루기로 소비자 보상 어려움(MRT 특정 아님, 숙소 엣지 설계 근거). |

### B′-3. 측정자 채점 가이드 (⚠ baseline과 개수·성격이 다름 — 반드시 읽을 것)

> 경계 케이스(사후↔여행 전, 확정지연↔기타, 취소정책↔환불지연)는 [축 정의 경계 가이드](../voc/axis-guide.md)로 판정한다. baseline은 이 가이드 없이 측정됐으므로, 파일럿 정확도 상승분에 가이드 효과가 섞임을 해석에 명시.

정답 축(voc-025~033):

| # | 여정 | 마찰 | 심각도 |
|---|---|---|---|
| 25 | 여행 중 | 긴급 대응 | 상 |
| 26 | 탐색 | 리뷰 신뢰성 | 상 |
| 27 | 사후 | 리뷰 신뢰성 | 중 |
| 28 | 사후 | 리뷰 신뢰성 | 중 |
| 29 | 사후 | 리뷰 신뢰성 | 하 |
| 30 | 예약 | 기타 | 중 |
| 31 | 여행 전 | 기타 | 중 |
| 32 | 여행 중 | 긴급 대응 | 하 |
| 33 | 여행 중 | 긴급 대응 | 중 |

집계: (마찰) 리뷰 신뢰성 4 · 긴급 대응 3 · 기타 2 / (심각도) 상 2 · 중 5 · 하 2.

**두 가지 비교 주의:**
1. **개수 불일치**: baseline 24건 vs 파일럿 9건. 분류 정확도는 **raw count가 아니라 비율(%)로 비교**한다. (baseline 3축 12.5/24 ≈ **52%** ↔ 파일럿 x/9)
2. **성격 불일치**: 파일럿 세트는 **리뷰 신뢰성이 4건(44%)으로 지배적**인데, baseline 24건에는 리뷰 신뢰성이 **0건**이었다. 즉 파일럿 diagnose는 baseline이 다루지 않은 축을 주로 테스트한다 — 정확도 비교 시 이 축 편향을 감안하고, 순수 동형이 아님을 결과에 명시.

## 방법론 메모 (측정자)

- **과제 A′는 강한 동형**(구조·난이도 일치, 목적지만 교체) → 시간·품질 비교가 방정하다.
- **과제 B′는 약한 동형**: 개수(9 vs 24)·마찰 성격(리뷰신뢰성 지배)이 baseline과 달라 정확도 직접 비교의 신뢰도가 낮다. 옵션:
  - (권장) 정확도를 **비율(%)**로만 비교하고, "리뷰신뢰성 축 편향" caveat를 붙인다.
  - 더 엄밀히 가려면 공개 소스에서 **신규 VoC ~15건을 추가 수집**해 baseline과 유사한 24건·유사 축 분포로 맞춘다(수집 방식은 [`voc_samples.json`](../../fixtures/voc/voc_samples.json) `_meta.collection_method` 준수).
- A안 방침상 **과제 B는 품질(분류 정확도)만** 판정하므로, B′의 시간은 참고용으로만 기록한다.
