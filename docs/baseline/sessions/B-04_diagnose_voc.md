- 과제: B(diagnose)
- 참가자: PE-2
- 제출일: 2026-07-07
- 사용 도구: TEL `/mrt diagnose` **미사용**. 실시간 인벤토리 미사용 — "요약문만 기준으로 판단". **순수 baseline 조건 충족.**

## 문제 인식

핵심 리스크는 "가격"보다 **확정 이후 실제 이용 가능성**. 이 페르소나(부모님 동행)에게 예약 확정은 상태값이 아니라 발권 완료·공급사 확정·미팅정보 확인·현지 연락 가능·취소 비용 이해까지 묶인 신뢰 문제.

## 반영해야 할 VoC 클러스터 (24건 전량 커버)

| 클러스터 | 포함 VoC | 요지 |
|---|---|---|
| **확정 공백** | #9~14 | 확정 메일 지연·공급사 무응답·사후 취소·자동발권 실패 → "결제했는데 정말 갈 수 있나" 불안. 부모님 동행은 대체 일정 확보 부담이 커 치명적. |
| **현지 긴급 연락** | #5~8, #24 | 가이드 연락처 접근 지연·미팅정보 불일치 시 고객지원 대기만 남는 구조. 야시장 버스투어는 미팅 장소/시간은 확인됐으나 예약 전 가이드 긴급 연락처 미확인 → "확정 후 언제 연락처가 열리는지" 명시 필요. |
| **취소/환불 예측 불가** | #4, #15~23 | 카테고리별 환불 규정·철회 불가·발권/여행사 수수료·바우처 환급은 "무료취소" 배지로 해소 안 됨. 실수 클릭·일정 변경 두려움. |
| **CS 접근성 병목** | #1~3, #7~8 | 전화·채팅·메일 모두 지연 → 위 세 리스크 증폭. 해결책은 상담 연결 개선만이 아니라 상담 전 자기 해결 상태 확인/대체 경로 제공. |

> #7·#8은 '긴급 연락'과 'CS 접근성' 두 클러스터에 중복 배치 — 증폭 요인으로 재사용(분석상 의도된 중복).

## 실험 브리프 보강안 (B-03 실험 확장)

기존 `예약 확정 체크포인트 카드` → **"이용 가능성 보증 카드"**로 확장.

- **가설**: 항공·숙소·투어 결제 전/후 화면에 `확정 단계`·`현지 연락처 공개 시점`·`미팅 정보`·`취소 시 실제 비용`·`긴급 대체 경로`를 한 카드로 보여주면, 부모님 동행 여행자의 예약 전 이탈과 확정/환불/연락 CS 문의가 감소.
- **핵심 지표**: 예약 전환율, 결제 후 24h 내 확정 관련 문의율, 출발 72h 전 미팅/연락처 문의율, 취소/환불 정책 문의율.
- **QA**: `공급사 확정 대기`·`자동발권 실패`·`가이드 연락처 미노출`·`취소요청 철회 불가`·`모바일 좁은 화면 수수료/환불방식 누락 없음` 케이스 포함.

## 채점 메모 (측정자 참고)

- **분류 정확도 미산출**: 24건을 여정×마찰×심각도로 전량 분류하지 않고 주제 클러스터로 묶어, B-02식 3축 완전일치율(예: 13/24) 대조가 불가. 클러스터-마찰 대응은 대체로 정합(확정 공백↔확정 지연, 취소/환불↔취소 정책+환불 지연). 표본 정확도 비교 시 B-02와 형식이 달라 **직접 비교 주의**.
- **타임스탬프 전체 미기록** → 시간 지표 기여 없음(B-03과 동일 문제).
- **순수 baseline 조건 충족**(인벤토리 미사용) — B-02와 동일 조건.
- **within-subject 연계**: PE-2의 A(B-03)→B(B-04)가 서로 참조됨. 독립 diagnose가 아니라 시나리오 보강 성격이라, "diagnose 단독 소요·품질"을 재려면 이 연계 편향을 감안.
