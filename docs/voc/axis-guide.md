# VoC 분류 축 정의 — 경계 판정 가이드

- 작성일: 2026-07-09
- 목적: baseline 진단 세션(B-02·B-04·B-06)에서 공통으로 흔들린 경계 3곳을 판정 규칙으로 못박아, 파일럿 채점 노이즈와 `/mrt diagnose` 출력 편차를 줄인다.
- 규칙 도출 근거: [`fixtures/voc/voc_samples.json`](../../fixtures/voc/voc_samples.json)의 정답 축 33건(voc-001~033) 패턴을 성문화한 것 — **기존 정답 축과 모순 없음**.
- 사용처: 파일럿 채점([`pilot_task_briefs.md`](../baseline/pilot_task_briefs.md) B′) + `/mrt diagnose` 처리 규칙([`commands/diagnose.md`](../../commands/diagnose.md)).

## 축 어휘 (정본)

- **여정 단계**: 탐색 / 예약 / 확정 / 여행 전 / 여행 중 / 사후(환불/CS)
- **마찰 유형**: CS 단절 / 확정 지연 / 환불 지연 / 리뷰 신뢰성 / 취소 정책 / 긴급 대응 / **기타**
- **심각도**: 상 / 중 / 하

> ⚠ 마찰 유형에 **"기타"가 정식 축 값**이다(voc-030·031이 기타). "기타 없음" 전제로 억지 분류하지 말 것.

## 공통 원칙

1. **여정 단계 = 마찰이 여행자에게 발생한 시점** (문의 주제나 상품 유형이 아님).
2. **마찰 유형 = 무엇이 잘못됐는가의 1차 원인** 1개. 복수 해당 시 여행자 피해가 가장 큰 축 1개만.
3. 건당 셀 1개씩 — 한 건을 두 단계/두 유형에 걸치지 않는다.

## 경계 규칙 3건 (baseline에서 흔들린 곳)

### R1. 사후 ↔ 여행 전 (CS 단절 계열)

- **"사후(환불/CS)"는 이용 종료 후만이 아니라 취소·환불·CS 처리 국면 전체를 포함한다** (축 이름 그대로). 여행 전에 일어난 취소라도 취소·환불 국면이면 사후다. (예: voc-018 숙소 예약 취소 91% 공제 → **사후**)
- 시점 맥락이 없는 CS 접근성 일반 불만(전화 미연결, 채팅 지연, 이메일 무응답)은 CS 수요가 집중되는 **사후**로 귀속한다. (voc-001~004)
- 단, 요약에 시점이 명시되면 그 시점을 따른다: "사전 확인/여행 준비" → **여행 전**(voc-005), "여행 중/현지" → **여행 중**(voc-006~008).

### R2. 확정 지연 ↔ 기타 (+ 예약 ↔ 확정 단계)

- 마찰의 결과가 **"확정이 늦어진다"로 서술되면 확정 지연**. 결제·시스템 오류 자체가 마찰이고 확정 지연 서술이 없으면 **기타**. (voc-013 결제 오류→자동발권 실패→확정 지연 = 확정 지연 / voc-030 할인코드 결제 오류 = 기타)
- 단계 판정: 마찰 발생 지점이 **예약 행위(입력·결제) 자체면 "예약"**(voc-013·020·030), **결제 완료 후 확정 통보 대기 구간이면 "확정"**(voc-009~012·014).

### R3. 취소 정책 ↔ 환불 지연

- **속도의 문제 = 환불 지연**: 환불 결정에는 다툼이 없고 리드타임·처리 기간이 긴 것. (voc-015~017)
- **규정·조건·금액의 문제 = 취소 정책**: 공제율, 바우처 환급, 철회 불가, 수수료 구조, 카테고리별 규정 상이, 결정 불이행. (voc-018~023)
- 한 건에 둘 다 보이면: 돈을 못/덜 받는 구조 문제가 있으면 취소 정책, 전액 받는데 늦기만 하면 환불 지연.

## 채점 적용 (측정자)

- 참가자 분류가 이 가이드와 달라도 **근거가 타당하면 감점하지 않고 별도 메모** — task_briefs B-3 채점 원칙 유지.
- baseline(B-02·B-06) 정확도는 이 가이드 없이 측정된 값이므로, 파일럿 정확도가 오르면 "가이드 효과 + 플러그인 효과"가 섞여 있음을 결과 해석에 명시한다.
