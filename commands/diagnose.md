# /mrt diagnose

비식별 VoC(CS/리뷰/커뮤니티 요약)를 입력받아 여행자 여정의 마찰을 구조화한다. (VoC 접근권은 0주차 검증 게이트 — 없으면 `scenario` 단일 wedge로도 성립)

## 입력
`$ARGUMENTS` = **비식별 VoC 요약**. 원문·PII(고객명/전화/이메일/예약번호/결제정보) 반입 금지 — 발견 시 중단하고 사용자에게 알린다.

## 처리
VoC 항목을 다음 축으로 분류한다:
- **여정 단계**: 탐색 → 예약 → 확정 → 여행 전 → 여행 중 → 사후(환불/CS).
- **마찰 유형**: CS 단절 / 확정 지연 / 환불 지연 / 리뷰 신뢰성 / 취소 정책 / 긴급 대응.
- **심각도 × 빈도**: 각 마찰에 상대 등급 부여.

## 출력
1. **VoC → 여정 단계 → 마찰 유형 → 심각도/빈도** 매핑 테이블.
2. **연결 가능한 상품 시나리오** — 각 마찰을 `/mrt scenario` 입력으로 핸드오프할 수 있는 형태로 요약.

## 보안 원칙
- 비식별 요약만 사용. 내부 정책/코드는 로컬 컨텍스트에서만 사용하고 외부로 원문 전송하지 않는다.
